# Kehrwoche

Telegram-бот для еженедельных напоминаний об уборке туалета в WG.

## Как работает

Каждый четверг в 09:00 бот отправляет в группу кто дежурит на этой неделе.
Ротация — по кругу через список комнат из `ROOMS`.

Расписание хранится в `schedule.json` на диске. При старте:
- файл есть → читает и работает
- файла нет → генерирует N недель и сохраняет

Когда расписание закончится — удали `schedule.json` через shell панели JustRunMy.App, перезапусти бот. Сгенерирует снова с правильной следующей комнатой (продолжает с того места где остановился).

## Команды в Telegram

| Команда | Что делает |
|---------|-----------|
| `/wer` | Кто дежурит на этой неделе |
| `/plan` | Расписание на 4 недели вперёд |

## Структура

```
main.go                  — точка входа, Thursday-крон, авто-генерация
internal/
  config/config.go       — читает конфиг из env vars
  schedule/schedule.go   — логика расписания (генерация, поиск, форматирование)
  bot/bot.go             — Telegram polling, команды, отправка
Dockerfile               — multi-stage build (alpine, non-root)
```

## Env vars

| Переменная | Обязательная | Описание |
|------------|-------------|----------|
| `TELEGRAM_BOT_TOKEN` | да | Токен от @BotFather |
| `ROOMS` | да | Комнаты через запятую в порядке ротации |
| `CHAT_ID` | да | ID Telegram группы |
| `SCHEDULE_WEEKS` | нет | Сколько недель генерировать (default: 8) |
| `SCHEDULE_PATH` | нет | Путь к файлу расписания (default: schedule.json) |
| `TZ` | нет | Часовой пояс (default: `Europe/Berlin`) |
| `SEND_NOW` | нет | Установить `1` чтобы разово отправить напоминание и выйти (используется в `make send`) |

Пример:
```
TELEGRAM_BOT_TOKEN=1234567890:AAF...
ROOMS=Zimmer 1,Zimmer 2,Zimmer 6,Zimmer 8
CHAT_ID=-1001234567890
```

## Локальная разработка

```bash
cp .env.example .env   # заполнить токен и chat_id
make run               # запустить бот (polling)
make send              # разово отправить напоминание и выйти
make build             # собрать бинарник в out/main
make test              # тесты
make fmt               # форматирование
```

## Деплой (JustRunMy.App)

Git push → платформа билдит Docker образ → запускает.

1. Добавить remote:
```bash
git remote add deploy <url от JustRunMy.App>
```
2. Установить env vars в их панели
3. Пушить:
```bash
git push deploy main
```

При первом старте `schedule.json` создаётся автоматически. Env vars из панели — не нужен `.env` файл на сервере.

## Нюансы

**schedule.json не в git** — генерируется на сервере при старте. Если диск эфемерный (сбрасывается при редеплое) — файл пересоздаётся автоматически, ротация продолжается правильно.

**Ротация при перегенерации** — `Extend()` смотрит на последнюю запись в файле и продолжает с следующей комнаты. Цикл не сбивается.

**ISO week** — расписание хранится в формате `2026-W25`. Год указан явно, чтобы не было путаницы на стыке лет (W52 2026 ≠ W52 2027).

**Часовой пояс** — в Docker образе установлен `tzdata`. Контейнер по умолчанию работает в UTC. Если нужно 09:00 по берлинскому времени — добавить `TZ=Europe/Berlin` в env vars.

**Polling, не webhook** — бот опрашивает Telegram каждые 60 секунд. Для еженедельного напоминания достаточно, webhook не нужен.
